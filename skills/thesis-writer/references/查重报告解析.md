# 知网查重报告 PDF 转 Markdown

将知网（CNKI）查重报告 PDF 转 Markdown 时，多种工具效果差异很大。下表为实测对比（以《改革开放以来高校辅导员职能的演变》三份报告单为样本）。

## 方案对比

| 问题 | MinerU (cloud VLM) | pandoc (`pdftotext -layout`) | pdf skill (pdfplumber) | 网页 HTML |
|---|---|---|---|---|
| 百分号变 LaTeX | ✗ | ✓ | ✓ | ✓ |
| 9 章节标题齐全 | ✗ 仅 5/9 | ⚠ 9/9 但为列表项 | ✓ 9/9 `##` | ✓ |
| 首页柱状图数值重复 2 次 | ✗ | ✗（PDF 渲染本身就重复） | ✓ 已去重并转表 | ✓（HTML 中本来就只有一份数据） |
| 表格为标准 Markdown（GFM） | ✗ HTML `<table>` | ⚠ 空格对齐文本 | ✓ 标准 `|---|` | ✓ |
| 全文对照左右双栏区分 | ✗ 混乱 | ✗ 混乱 | ✗ 左右栏被聚到同一行 | ✓ DOM 结构天然分开 |
| 引用标注（红/绿色块）保留 | ✗ | ✗ | ✗ | ✓ `Font_Color_Red`/`Font_Color_Green` 类名 |
| 相似来源元数据（作者/期刊/日期/是否引证） | ⚠ | ⚠ | ⚠ | ✓ 完整 |

**结论**：
- **简洁报告单 / 全文标明引文报告单**：pdfplumber 方案足够，自动化好（脚本见 `scripts/查重_pdf_to_md.py`）。
- **全文对照报告单**：必须用 HTML 方案。脚本 `scripts/查重_html_to_md.py` 已实现，效果验证良好（112KB md，9 章节齐全，所有相似源完整、红/绿色标保留）。

## HTML 方案使用流程

1. 在浏览器打开知网查重报告页（`https://cx.cnki.net/...`）。
2. F12 打开 DevTools → Elements，找到 `<div id="detectionResult">`。
3. 右键 → Copy → Copy outerHTML。
4. 保存到 `<报告集>/html/<报告名>.html`。
5. 运行：
   ```bash
   uv run scripts/查重_html_to_md.py <html_file> -o <out.md>
   ```

脚本含相邻同色高亮自动合并（处理 `<span class="Font_Color_Red">A</span><span class="Font_Color_Red">B</span>` → `**【红】AB**`）和空高亮过滤。

## pdfplumber 方案核心思路

1. **过滤水印**：`non_stroking_color == (2.0,)` 的灰度字符是水印；字号 > 20 也基本是装饰大字。
2. **按 y 坐标聚行**：相同 `top`（容差 ±2.5）的字符归为一行，按 `x0` 排序拼接，相邻字符 x 间距 > 0.5 字号时插入空格。
3. **按字号判定层级**：≥15 为 `#`，11.5–15 为 `##`，其余为正文段落。
4. **柱状图特殊处理**：用正则 `(\d+%)\((\d+)\)` 重复匹配并去重，统一输出为 GFM 表格。

## 已知问题：双栏页文本错行

**全文对照报告单** 第 3 页起为左右双栏对照（左列原文 / 右列相似来源）。pdfplumber 按 y 坐标聚行时不区分 x，会把同一行的左右两列文字粘在一起：

```
展。习近平总书记在全国高校思想政治工作会议上指出      ← 仅左列
"要建设好辅导员队伍，保证这支队伍高质量、高水准         ← 仅左列
，保证这支队伍后继有人、 。从软指标出发提出的高质量要求  ← 左+右拼成一行 ❌
```

### 这种 md 渲染和阅读会怎样？

- **Markdown 渲染**：仍然是合法 Markdown，渲染器（VS Code、GitHub、Obsidian 等）会正常显示，不会报错；只是看起来文意不通。
- **AI 阅读**：LLM 可以读但语义已被破坏——左右两列内容互不相关却被串成一句，语序颠倒、半句被截断，模型容易做出错误归因（"原文说 X 是因为 Y" 实则 X、Y 来自两份不同文献）。**不宜直接拿来做引用核查**。
- **人眼阅读**：很难看，需要对照原 PDF 才能还原。

### 修复方向

- **首选**：改用 HTML 方案（DevTools 复制 DOM）。
- 备选：按页面宽度中线（约 `page.width/2`）将字符分为左右两组，分别 `group_lines` 后用 `|` 拼接为两栏 GFM 表，或先输出左栏再输出右栏。

## HTML 方案 DOM 结构要点

知网网页报告单的关键 DOM 节点：

- `#detectionResult` — 整个报告容器
- `.eBox` / `.profileBox` / `.segmentedDetectionResultsBox` — 顶部三大块（复制比 / 结果分布图 / 分段检测）
- `#detailBox30` ~ `#detailBox38` — 9 个章节的对照详情
- 每个 `.ACertainParagraph` 内：
  - `.ACertainParagraphLeft` — 论文原文（含 `.title` 显示"此处有 N 字相似"）
  - `.ACertainParagraphRight` — 相似来源列表，每条为 `.ACertainParagraphRightBox`，含 `.articleTitle` 和 `.articleContent`
- 高亮 span：
  - `.Font_Color_Red`（class 名是 `red_font`）— 文字复制部分（未标引用）
  - `.Font_Color_Green` — 文字复制部分（已标引用）
- 引用状态：相似来源末尾的 `（是否引证：是/否）`

## 名词

- **GFM**：GitHub Flavored Markdown，CommonMark 的扩展。常用扩展包括管道表格（`| a | b |`）、删除线（`~~x~~`）、任务列表（`- [ ]`）、代码块语言标注、自动链接等。本论文仓库内统一使用 GFM 表格语法。
