# PR 工作流

## Git 初始化

每轮在新容器里：

1. `gh auth setup-git` — git 走 HTTPS + `GH_TOKEN`
2. git user 身份通过 `GIT_AUTHOR_NAME` / `GIT_AUTHOR_EMAIL` 环境变量注入

## 代码仓库

每轮重新 clone，不持久化：

```bash
gh repo clone <OWNER>/<REPO> /root/workdir/<REPO>
```

首次进入检查 `autopilot-work` 分支存在性，不存在则 abort + 通知用户手动创建。

## 分支策略

- base 分支：统一 `autopilot-work`（所有仓库一视同仁）
- feature 分支：`autopilot/<维度>-<YYYYMMDD-HHMM>`
- 从远程 base 切出：`git checkout -b autopilot/<dim>-<时间戳> origin/autopilot-work`

## Owner 预检

提 PR 前验证 `gh repo view --json owner -q '.owner.login'` 与预期一致，不一致时 abort + 告警。

## PR 创建

```bash
gh pr create \
  --repo <OWNER>/<REPO> \
  --base autopilot-work \
  --head autopilot/<dim>-<时间戳> \
  --draft \
  --title "<维度>: <描述>" \
  --body "<正文>"
```

## PR 正文模板

```
## What

<改了什么>

## Why

<为什么>

---

Generated by Hermes Agent (autopilot runbook)
- provider / model: <运行时填入>

Human review required before merge.
```
